API Fetch & Save Script

Fetch JSON data from any public API URL, understand its structure, and get a ready-to-run script that saves timestamped records to a local file.

Subscription required — This sample runs inside Claude.ai using the Cowork feature. A Pro or Team subscription with Cowork access is required. No API key or local Python install needed.

What this does

Fetching data from a public API and saving it locally involves understanding the JSON structure, adding timestamps, and handling errors gracefully. This task uses Claude to:

  • Analyse the JSON response from any URL you provide
  • Explain the data structure in plain English
  • Generate a complete Python script that fetches and saves timestamped records
  • Optionally add polling (run every N seconds) and append new records to the same file

You can paste in a raw JSON response, or let Claude use the provided example from sample_data/example_output.json.

Step-by-step in Claude Cowork

  1. Open claude.ai and sign in to your Pro or Team account.
  2. Click Cowork in the left sidebar to open a Cowork session.
  3. Upload sample_data/example_output.json from this repo (or paste a JSON snippet from your own API directly into the chat).
  4. Paste the prompt below into the chat and press Enter: 
    I've uploaded a JSON file (or pasted a JSON response) from an API endpoint.

Please:

1. Analyse the JSON structure and describe what data it contains in plain English (field names, data types, nesting).
2. Identify the most useful fields to extract and display in a human-readable summary.
3. Write a complete Python script (standard library only — no third-party packages) that:
   a) Fetches JSON from the URL: https://jsonplaceholder.typicode.com/todos/1
   b) Wraps it in a record with: fetched_at (UTC ISO timestamp), url, status_code, data
   c) Loads any existing records from data.json (as a list), appends the new record, and saves back to data.json
   d) Prints a clean summary of the fetched data to the terminal
   e) Accepts a --url flag to fetch from a different endpoint
   f) Accepts an --interval N flag to keep polling every N seconds (Ctrl+C to stop)
4. Show a sample of what data.json will look like after two fetches.

Output the script as a Python code block.
  5. Claude will explain the JSON structure, then output a complete, ready-to-run Python script.
  6. Copy the script, save it as fetch.py, and run it with python fetch.py. Modify the URL as needed.

Sample output

**JSON structure analysis:**
The response is a flat object with 4 fields:
- userId (integer) — the user who owns this todo
- id (integer) — unique todo identifier
- title (string) — the task description
- completed (boolean) — whether the task is done

Reference: local Python version

The Python script main.py in this folder is the complete fetch-and-save implementation (standard library only):

# Single fetch (saves to data.json):
python main.py

# Fetch from a custom URL:
python main.py --url https://api.coindesk.com/v1/bpi/currentprice.json

# Poll every 30 seconds:
python main.py --url https://jsonplaceholder.typicode.com/posts/1 --interval 30
Note: The Python script requires local file access and Python 3.8+. The Cowork approach above generates a customisable script you can adapt for any API without needing local tools.
